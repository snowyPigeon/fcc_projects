Introduction

This document introduces the key concepts and elements of coding with HTML (Hypertext Markup Language).

HTML is the most basic building block on a web page. HTML provides the structure and meaning of the the page. Other technologies are used with HTML to describe style, and to extend the functionality of the page. The hypertext refers to the links used on pages which connect with each other.

By the end of the lesson you will be able to:

  • Display text and images on a web page
  • Use separate text elements to bring some basic style to your page
  • Use sections and divs to group your content
History

In the early days of the Internet there was not an easy way to retrieve information. Sir Tim Berners-Lee, a software engineer working at CERN, the large particle physics laboratory near Geneva, Switzerland, created a system for presenting information and connecting pages of information together.

In March 1989, Berners-Lee made an initial proposal for what would become the web in his document Information Management: A Proposal.

In September 1990, Berners-Lee began working on the web.

Berners-Lee wrote "three fundamental technologies that remain the foundation of todays' web ...:

  • HTML: HyperText Markup Language. The markup (formatting) for the web.
  • URI: Uniform Resource Identifier. A kind of “address” that is unique and used to identify to each resource on the web. It is also commonly called a URL.
  • HTTP: Hypertext Transfer Protocol. Allows for the retrieval of linked resources from across the web."

World Wide Web Foundation, 2019

Basic Markup Formatting

HTML uses markup to describe how text and other elements should be displayed on a page in a Web browser. HTML markup includes special elements such as <head>, <body> and <div>. The majority of elements have an opening and closing tag. A complete pair of tags will look like this: <p></p>. The tag name is enclosed in the first tag and a forward slash precedes the tag name in the closing tag. Some tags are self closing such as the <img> tag which is used with a file address for enclosing a link to an image, like in this example: <img src="http://myhomepage.com/images/cat_photo.jpg">.

The Head of a Document

The <head> of a document is used for linking to related files, scripts, page window titles and meta-data that the author does not want output to the browser display.

A typical web page will have a <head> section similar to below:

<head>
<meta charset="UTF-8">
<meta name="description" content="HTML Introduction">
<meta name="keywords" content="HTML">
<meta name="author" content="A Name">
<link href="https://fonts.googleapis.com/css?family=Chivo|Titillium+Web&display=swap" rel="stylesheet">
<link rel="stylesheet" type="text/css" href="homepagestyle.css">
<style>
p, ul, a { font-family: Arial, sans-serif; }
<style>
<title>My Homepage</title>
</head>

The <head> element contains some meta-data, a link reference to an external font source, a link to an external style sheet, an internal style sheet, and the title for the browser window. A font source such as provided by Google can allow you to use a wider range of font faces than those interpreted by a browser by default.

The Body of a Document

The <body> of the document contains the code for the elements that you wish to be visible. This is where you would insert your content such as text, headings and images.

A very simple example:

<body>
<h1>Welcome</h1>
<p>Hello World!</p>
</body>

In the example above, the web page contains a heading at the largest size - in h1 tags, and some smaller text wrapped in paragraph tags.

Headings

Headings on a web page work like titles and sub-titles in a book. In HTML a hierarchical system is used for the main heading and sub headings which decrease in size according to their importance. You should use the h1 tag for your main heading, and you should use h2 tags for your headings of next level importance, and so on.

In the example below we present the code for the heading and show how it will look without any additional formatting:

The Body

<h2>The Limbs</h2>

The Hands

The Fingers

<h5>The Finger Joints</h5>

Paragraphs

Most of your text will be enclosed within <p></p> tags. Open your text with the opening <p> tag and close your text with the closing </p> tag. Again, we shall demonstrate how the code appears and how the text should display on the page.

<p>My first paragraph. This could be a single sentence or it may span several sentences.</p>

My first paragraph. This could be a single sentence or it may span several sentences.

<p>My second paragraph. This could be a single sentence or it may span several sentences. This will start on a new line.</p>

My second paragraph. This could be a single sentence or it may span several sentences. This will start on a new line.

Images

Images can enhance the overall appearance of a page and they can help to break up the text based content.

If you are working purely on your home computer and have some image files in the same folder as your web page file, you can assign a relative pathname to an image tag, and when you view the html (web page) file in a browser the image should display. If you are working with an online file, then you will need to upload your images as well. If the images and html file are in the same folder, again, you can use a relative link. If your images are hosted somewhere else, then you will need to include the whole path or absolute pathname in order for the image to display.

Here is an example, with the code on the left and the output on the right:

<img src="https://live.staticflickr.com/65535/
50315642467_0beb1d4d81_z.jpg" style="display:block;width:150" alt="Inside Cover of Arithmetic Book">

We have added some inline (within tag) CSS (Cascading Style Sheets) formatting after the file address in order to control the size of the image displayed.

In this case, an absolute pathname has been specified for the image address because it exists in a different location to the html file for this page. A relative pathname would be of the form "filename.filetype" or possibly "/foldername/filename.filetype". An absolute pathname will include the whole web address for the file, including https or http.

Another point of note is the alt text provided within the image tag. It is good practice to include alt text to describe what the image shows, for the benefit of users using screen readers or in case the image does not display.

Named Sections and Divs

You can control the structure of your page using <section> and <div> tags wrapped around your content. There are additional section elements with more specific names: <nav>, <article>, and <aside>. Note that <nav> is used only as a navigation element and a <nav> cannot be nested within another <nav>.

Here is an example, with the code on the left and the output on the right:

<section><p>This section contains some informative text. This section contains some informative text. </p></section>

This section contains some informative text. This section contains some informative text.

<div><p>This div contains some informative text. This section contains some informative text. </p></div>

This div contains some informative text. This section contains some informative text.

Sections and divs can be arranged to sit side by side, in columns, in rows, or within a grid. In modern browsers, the placement of these elements is organised with CSS which is beyond the scope of this lesson.
